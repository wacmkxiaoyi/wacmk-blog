# Access Rules and Sharing

## The Three Main Access Modes

Articles, books, and attachments can use one of three main visibility modes:

- `Public`
- `Private`
- `Conditional`

## Public

Public content is immediately available unless another rule in the surrounding context changes the user experience.

## Private

Private content is normally restricted to:

- the owner or author
- privileged staff users

## Conditional

Conditional content requires one or more rules to be satisfied before access is granted.

## Supported Condition Types

Depending on the object type, the site can use these condition types:

- money
- points
- encrypted password
- book-only for articles

## Very Important Difference: Money vs Points

### Money

Money-based content can require a purchase action.

If the user has enough money balance, the site can offer a purchase button to unlock the content.

### Points

Points-based content does not usually spend points.

Instead, it checks whether the user already has enough points.

That means:

- points are usually a threshold
- points are not usually deducted to unlock content

## Password-Protected Content

Password-protected content uses verification rather than a balance check.

If the password is correct, the unlock is remembered for that user and object.

## Book-Only Content

Book-only is not the same thing as private.

It means the article is intended to be opened inside a book context rather than as a normal standalone article.

## Access Gate Page

When content is protected, the platform may show an access verification page or gate.

This gate can show:

- each condition type
- whether the condition is already satisfied
- whether the next step is password verification or purchase

## Unified and Standalone VIP Access

### Unified

VIP users and non-VIP users use the same content rule set.

### Standalone

VIP users can use a separate VIP rule set for the same content.

This can change:

- visibility behavior
- required conditions
- how difficult or easy it is for VIP users to access the content

## VIP Discounts

VIP users can also receive discounted money or points requirements depending on site configuration.

These discounts are not limited only to standalone content.

## Share Links

Share links are limited to public, share-eligible content.

Protected conditional content does not turn into freely readable content just because a share link exists.
