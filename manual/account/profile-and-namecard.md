# Profile and Namecard

## Profile Sections

The profile workspace is split into multiple sections:

- Basic information
- Account security
- Money history
- Points history
- User group
- My articles
- My books
- My attachments
- My comments

## Basic Information

In the basic section, users can manage:

- avatar
- display name
- gender
- age
- personal description
- GitHub link
- website link
- Twitter link
- QQ
- whether the email is shown on the public namecard

## Avatar Behavior

Users can:

- upload a new avatar
- mark the current avatar for removal
- undo the removal before saving

## Account Assets

The profile and user menu may show two balances:

- money
- points

These values change when users receive rewards, unlock content, or are adjusted by administrators.

## Money History and Points History

The history pages explain how balances changed over time.

Examples of reasons include:

- daily login reward
- first comment reward
- author reward
- content purchase
- administrator adjustment

## User Group Section

The user group section shows a capability table comparing:

- normal user
- each configured VIP tier

This section explains feature availability and reward differences, but it does not change the account by itself.

## Public Namecard

Each user can have a public-facing namecard page that may include:

- avatar
- display name
- administrator badge
- VIP badge
- money and points balances
- gender and age
- description
- social links
- optional email visibility
- recent posts
- recent books
- recent comments

## Namecard Visibility Notes

- Email appears only if the user enabled it.
- Visible content on the namecard depends on what the viewer is allowed to access.
- Recent comments can appear with their related article or book context.
