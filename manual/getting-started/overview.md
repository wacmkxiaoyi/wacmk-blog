# Platform Overview

## What This Site Is

`wacmk-blog` is a content platform built around:

- articles
- books made from articles
- downloadable attachments
- profile and public identity pages
- comments, stars, and feedback
- optional VIP-based access and rewards

It can behave like a normal blog, a protected content library, or a mixed publishing and resource-delivery platform depending on how the site administrator configured it.

## Main Parts of the Site

### Authentication Area

This includes:

- login
- registration, if enabled
- password reset, if email delivery is configured

### Authenticated Main Area

After sign-in, users usually work inside these main areas:

- `Home`
- `Articles`
- `Books`
- `Tags`
- `Profile settings`

### Public Share Area

Some content can also be opened from public share links. These pages are different from normal authenticated pages and may restrict interaction.

## What Different Users Can Usually Do

### Regular Logged-In Users

Regular users may be able to:

- read accessible articles and books
- use tags and search
- comment and react to content
- manage their own profile
- manage their own posts, books, attachments, and comments if the site allows it

### VIP Users

VIP users may receive:

- additional publishing permissions
- upload permissions
- discounted content requirements
- access to standalone VIP rules
- extra rewards for login and commenting

### Staff and Superusers

Staff and superusers can access the management workspace through the user menu.

## Why the Same Content Can Behave Differently for Different Users

This platform does not rely only on simple public/private content.

The same object can behave differently based on:

- whether the viewer is the author
- whether the viewer is staff
- whether the viewer is VIP
- whether the object is public, private, or conditional
- whether the object uses unified or standalone VIP access
- whether the user already unlocked the content through password verification or purchase

## What the Manual Focuses On

This manual explains how the site behaves from the user side, including:

- where to find features
- what certain access gates mean
- why a button may or may not appear
- what happens when a user unlocks, buys, comments on, or shares content
